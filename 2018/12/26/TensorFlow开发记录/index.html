TensorFlow开发记录

Posted on 2018-12-26
环境安装：

a) TensorFlow 有CPU和GPU两个版本，电脑如果使用的是非集显的GPU最好是安装GPU版本，这样算力更高。（具体课查看NVIDIA官方的算力表，算力在3.0以上都推荐安装）

在Linux平台安装TensorFlow GPU版本需要NVIDIA GPU驱动（目前TensorFlow还不支持AMD），CUDA工具包和cuDNN的sdk

首先要安装GPU驱动，目前TensorFlow 支持CUDA9.0 ，GPU驱动支持384.x 及以上，

CUDA的版本过高或过低都需要额外的配置，所以推荐一一对应的安装TensorFlow和CUDA版本。

我安装驱动后执行nvidia-smi命令显示信息如下：

然后是CUDA的安装，我安装的时候Ubuntu18并未被正式支持，所以如果是在Ubuntu16进行安装的话会轻松很多，

如果在Ubuntu18版本进行安装可能要使用源码编译安装

Ubuntu16运行以下命令应该就能完成对CUDA的依赖安装

之后就是cuDNN,cuDNN需要安装安装7.2以上的版本

在~/.bashrc中添加环境变量

export LD_LIBRARY_PATH=”$LD_LIBRARY_PATH:/usr/local/cuda/lib64:/usr/local/cuda/extras/CUPTI/lib64”

export CUDA_HOME=/usr/local/cuda

至此，应该大致的依赖都装完了，然后利用python自带的pip包管理工具应该就可以就行TensorFlow GPU版本的安装了:

pip install tensorflow-gpu

我安装的版本如下，我并没有安装CPU版本的TensorFlow，但其实两个版本可以同时安装，后续使用的时候指定运算设备其实就可以了

tensorboard是训练中常用的一个可视化工具，最好也装上，后续在监测训练效果的时候是很常用的

数据集的采样：

数据集的采样其实可以用爬虫完成，但是为了保证训练集的正确性，我这里使用了他

人给出的一组奶酪通心粉图片，因为后续还有图像标注工作要做,而这个数据集是别人使用过的，可以保证数据的正确性，如果最后整个程序运行无误，那么可以替换掉数据集，再次训练，从而保证开发效率

数据集制作：

数据集的制作依赖于GitHub的开源项目 LabelImg，这是一个快速对数据集进行标注的工具，可以在Windows 和类Unix平台使用，对于每张图片，只要标注出目标并输入相应的标签就可以了，它会自动生成相应的xml文件记录标注信息，该工具需要qt环境

然后要安装一下Tensorflow Object Detection API

首先，克隆一下github的相关仓库：

git clone https://github.com/tensorflow/models.git

然后需要几个额外的库，安装一下：

sudo apt-get install protobuf-compiler python-pil python-lxml

然后用一下Protocol Buffers这个工具

在刚刚克隆的models文件夹下执行：

protoc object_detection/protos/*.proto –python_out=.

这个时候也可以在models/research 目录下执行

python3 setup.py install 以安装整个api
